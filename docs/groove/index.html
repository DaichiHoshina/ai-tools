/groove

YAML駆動の軽量マルチエージェントオーケストレーター

Claude Code内で完結 外部依存なし YAML定義

概要

grooveは、YAMLでワークフローを定義し、複数のAI Agentを協調実行する仕組みです。 各ステップでAgentを起動し、その出力結果に基づいて次のステップへ自動遷移します。 リトライ、タイムアウト、並列実行、フォールバックに対応。

使い方

/groove <task>                # ワークフロー自動選択
/groove <workflow> <task>     # ワークフロー指定
/groove --auto <task>         # 自動モード（完了後にcommit+push）
/groove list                  # ワークフロー一覧

タスクの内容からワークフローを自動選択します。「テスト」「TDD」を含む場合はtdd、「VSDD」「品質重視」はvsdd、それ以外はspec-drivenが選ばれます。

Agent一覧

spec-reviewer
タスク指示書の品質を検証。要件の明確性、変更対象の特定可能性を判定。
readonly

仕様レビュー通過後のタスクを実装。テスト・ビルド実行まで担当。

受入検査。5次元バイナリ評価（1つでもFAILなら全体FAIL）。

adversary
敵対的レビュアー。「欠陥がある」前提で5次元検証。

AIスロップ検出。過度な抽象化・不要パターン・冗長コードを3カテゴリで検出。

edit
simplifier
実装完了後のコード簡素化・整理。変更ファイルのみ対象。

実行エンジン

ステップ実行ループ

1. max_steps超過チェック → 超過ならABORT
2. loop_limit超過チェック → 超過ならABORT
3. Agent定義ファイルを読み込み
4. Agent起動（edit: bypassPermissions / readonly: default）
5. 出力から GROOVE_RESULT: {値} を抽出
6. rulesマッチングで次ステップを決定
7. step_count++, loop_count++ → 次ループへ

エラーハンドリング

設定型デフォルト説明
retryint0失敗時の最大リトライ回数
timeoutintなしステップのタイムアウト（秒）
fallback.nextstringなし全リトライ失敗時の遷移先

Agent失敗/タイムアウト時: retry上限まで再実行 → fallback定義があれば遷移 → なければABORT

並列実行

parallel指定されたステップは、複数Agentを同時起動します。結果の集約ルール:

結果条件
spec_issueいずれかがspec_issueを返した場合（最優先）
any_failいずれかがfailを返した場合
all_pass全Agentがpassを返した場合

配置と同期

groove/ はGit管理用のソースディレクトリ。実行時は ~/.groove/ が参照されます。

ai-tools/groove/
Git管理（ソース）
sync.sh to-local ↔ sync.sh from-local
~/.groove/
実行時に参照
./claude-code/sync.sh to-local    # リポジトリ → ~/.groove/
./claude-code/sync.sh from-local  # ~/.groove/ → リポジトリ
./claude-code/sync.sh diff        # 差分確認

パス解決の優先順位

優先度パス用途
1st.groove/プロジェクト固有のワークフロー
2nd~/.groove/グローバル（ホーム）

プロジェクト直下に .groove/ を置けば、そのプロジェクト専用のワークフロー・Agentを定義できます。

ワークフローYAML構文

name: my-workflow
description: ワークフローの説明
max_steps: 25          # 全体の最大ステップ数
loop_limit: 3          # 同一ステップの最大ループ回数

steps:
  - name: review
    agent: spec-reviewer     # agents/内のファイル名（.md除く）
    mode: readonly           # readonly | edit
    retry: 1                 # 失敗時リトライ回数
    timeout: 120             # タイムアウト（秒）
    rules:
      - on: pass             # GROOVE_RESULTの値
        next: implement      # 次のステップ名
      - on: fail
        next: ABORT          # 特殊値: ABORT, COMPLETE

  - name: implement
    agent: developer
    mode: edit
    rules:
      - on: done
        next: review_parallel
      - on: needs_input
        next: implement
        ask_user: true       # ユーザーに質問してから再実行

  - name: review_parallel
    parallel:                # 並列実行
      - agent: acceptor
        mode: readonly
      - agent: ai-reviewer
        mode: readonly
    rules:
      - on: all_pass
        next: COMPLETE
      - on: any_fail
        next: fix

  - name: fix
    agent: fixer
    mode: edit
    fallback:
      next: implement        # 全リトライ失敗時の遷移先
    rules:
      - on: done
        next: review_parallel
