核心結論
  1. churn pattern マッチは frustration を大幅に過大計上していた。実メッセージを分類すると、真の frustration churn はごく僅か。当初の「修正/直して +48%・違う +73% 悪化」は計測ノイズで、frustration 増ではなかった。
  2. 「長文冒頭結論 rule」(2026-05-22 追加) の効果は限定的だが副作用なし。つまり？/どういうこと？ は 1.3→1.12/d とほぼ横ばい。ただし真の どういうこと？ は誤検出を除くと 0.38/d まで下がり、rule は実は効いている可能性が高い。
  3. 最大 pattern 再度/もう一度 (5.5/d) の大半は正常な反復ワークフロー (jp-writing 修正サイクル・PR コメント再読込・SQL 再実行)。frustration ではない。

3. 真 churn 再集計 (ノイズ除去後)

5. 改善 (採用済 3 件)

① churn 計測 script の noise 除去
疑問文 (「何が〜と違う?」)・正常反復 (「再度出力/チェック」)・自己引用を churn から除外する filter を実装。永続化先 claude-code/scripts/。次回以降の計測精度を担保し、誤トレンド (+48%/+73%) を防ぐ。
② 「どういうこと？」射程拡張
残存 3 件は単発短文。次回 retrospective で直前 assistant 応答とのペア分析を行い、結論先出しを怠ったケースを特定 → rule 補強。今回は射程外ケースの存在を確認するに留める。
