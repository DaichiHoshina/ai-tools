# 読み物系文章の構造原則 (narrative writing)

blog / エッセイ / インタビュー記事 / 物語型の長文など、読み手が体験として読む文章の構造原則。

**適用範囲の宣言 (最重要)**: 技術文書 (PR / commit / RCA / DD / issue / chat 報告) には適用しない。技術文書は結論先出し ([PRINCIPLES.md](PRINCIPLES.md) の PREP / inverted pyramid) が正だ。本 file の「結論を隠して配列する」操作はそれと正面から矛盾する。迷ったら「読後に読み手が判断・行動するなら技術文書、何かを感じ取るなら読み物」で分ける。

## 4 操作の全体像

| 操作 | 内容 | canonical |
|---|---|---|
| 1 設計 | 終点を先に固定し、円環を張る | 本 file |
| 2 選別 | 抽象の核 1 語と具体の事実だけ残す | PRINCIPLES.md「AI臭を消す3変換」 |
| 3 変換 | 直接表現を別の層に写す | 同上 + 本 file の読み物系差分 |
| 4 配列 | 認知の順序で並べ、転回で折る | 本 file |

## 操作 1: 設計 — 終点を先に固定する

書き始める前に「最後の 1 文で回収するモチーフ」を 1 つ決め、それを冒頭に伏線として埋める。本文はこの 2 点間の移動として書く。読後感は結末の 1 文単体ではなく、冒頭からの距離と回収で生まれる。

- 書く前に 2 点だけ決める。回収するモチーフ 1 つと、読後に残したい核概念 1 語だ
- 冒頭段落は導入ではなく伏線として書く。説明から始めない
- 結末は要約ではなく回収にする。冒頭の語や情景をもう一度、意味を変えて置く
- 核概念の 1 語は鉤括弧で地の文から浮かせ、その周りに具体描写を配置する

例 (自作):

> 冒頭: 「引越しのたびに、使い道の分からない鍵が 1 本だけ箱から出てくる。」
> 結末: 「次の引越しでも、あの鍵はきっと捨てられないままだ。」

本文がどこへ進んでも、この 2 文が呼応していれば文章は閉じる。

## 操作 4: 配列 — 認知の順序で並べる

事実の時系列や重要度順ではなく、読み手に辿らせたい認知の順 (誤解、転回、理解の 3 段) に並べ替える。書き手が先に答えを知っていても、読者が発見する順序で出す。

- 結論に相当する評価 (「良い」「好きだ」) を先に書かない。書き手が辿った認知の順で出す
- 転回点は短文の独立段落で物理的に折る (「しかし。」「そして。」)。1 本の文章で 1-2 回に留める。連発すると効果が消える
- 想定される反論は読者より先に自分で言い、観察事実で退ける
- 最後の接続 1 歩だけ読者に渡す。全部言い切ると体験でなく説明になる

## 操作 2・3 の読み物系差分

選別 (抽象語を数字と事例へ) と変換 (評価語を根拠へ) の実体は [PRINCIPLES.md](PRINCIPLES.md)「AI臭を消す3変換」が canonical だ。読み物系での差分は 2 点だけある。

- 感情は評価語でなく身体反応と情景に写す (「感動した」ではなく「最後の段落まで指が止まらなかった」)
- 数字は主張の裏付けでなく実在感のために置く。年数・回数・距離など検証可能な細部を段落に 1 つ入れる

文レベルの自然さ (語順 / 読点 / リズム / AI 臭の除去) は natural-japanese skill と PRINCIPLES.md「AI臭の根本」が担う。本 file は段落と文章全体の構造だけを扱う。

## 総括原理

書くとは「伝えたいこと」を紙面から除去し、読者がそれを自力で再構成するための最小の部品を、発見可能な順に配置する行為だ。設計は再構成の枠を、選別は部品の純度を、変換は部品の形を、配列は発見の順路を担う。

## 出力前 self-check (4 項目)

```
[ ] 冒頭のモチーフを結末で回収している (円環が閉じている)
[ ] 評価語 (良い / 素晴らしい / 感動的) が本文に残っていない
[ ] 結論に相当する 1 文を直接書いていない (読者が組み立てる部品だけ置いた)
[ ] 転回の短文段落が 1-2 回ある (ゼロは平板、3 回以上は乱発)
```
