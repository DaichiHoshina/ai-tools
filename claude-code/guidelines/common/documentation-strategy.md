# ドキュメント分類・保存先ガイドライン

> 体系設計（種別の役割分担・関係性・Bounded Context 分割・命名規則・レビュープロセス）は [`doc-organization-principles.md`](./doc-organization-principles.md) を参照。本ファイルは「どこに保存するか」「どう書くか」に特化。

## ドキュメント種別と保存先

| 種別 | 内容 | 保存先 |
|------|------|--------|
| ADR | アーキテクチャ決定記録（構造・非機能要件・技術選定の理由） | GitHub（リポジトリ内） |
| DesignDocs | 詳細設計（実装方針・コンポーネント構成・データフロー） | GitHub（リポジトリ内） |
| PRD | 要件定義（新機能・ステークホルダー合意） | GitHub（リポジトリ内） |
| ProductSpec | リリース済み機能の最新仕様 | GitHub（リポジトリ内） |
| TechnicalSpec | 実装済み機能の技術仕様 | GitHub（リポジトリ内） |
| Runbook | 頻繁に発生する業務手順（障害対応・データ作成等） | Wiki/Notion等 |
| 開発Tips | 環境構築・デバッグ・ハマりポイント | Wiki/Notion等 |

**原則**: 設計・仕様系はGitHub（コードと一緒にバージョン管理）、運用手順・TipsはWiki/Notion（検索・更新が容易）。

## どこに書くか判断フロー

```text
ナレッジを書きたい
├─ 設計判断・技術選定の理由 → ADR (GitHub)
├─ 実装の詳細設計 → DesignDocs (GitHub)
├─ 機能要件 → PRD (GitHub)
├─ リリース済み機能仕様 → ProductSpec (GitHub)
├─ 技術仕様 → TechnicalSpec (GitHub)
├─ 繰り返す運用手順 → Runbook (Wiki/Notion)
└─ 開発Tips・ハマりポイント → Tips (Wiki/Notion)
```

## AI対応ドキュメントルール

RAG検索精度を意識した構造化ルール。AI検索やembedding生成の品質に直結する。

### ページ構成

| ルール | 基準 |
|--------|------|
| 1ナレッジ=1ドキュメント | 1ページに「1つの問い+1つの答え」のみ |
| 文量 | 800-1,500文字/ページ目安。2,000文字超なら分割 |
| 見出し階層 | H1-H3まで。H4が必要なら内容が複雑すぎるため分割 |

### 文体・表現

| ルール | 詳細 |
|--------|------|
| 主語明示 | 「担当者は〜」「ユーザーは〜」必ず記述 |
| 指示語禁止 | 「これ」「それ」「上記」「前述」→ 具体的な名称に |
| 略語禁止 | 初出時はフルスペルで記述 |
| 判断基準 | 「適宜」「状況に応じて」禁止 → IF-THEN形式で明文化 |
| 日付 | YYYY/MM/DD |

### 視覚・構造

| ルール | 詳細 |
|--------|------|
| 画像 | 必ずテキストでも完結するよう言語化（AIは画像を読めない場合あり） |
| テーブル | セル結合禁止、空欄は「なし」と記入 |
| 色による識別 | 廃止。見出しで区分けする |
| 臨時運用 | タイトル先頭に【臨時運用】+ 有効期限 or 解除条件を冒頭に記載 |

### 公開前チェックリスト

- [ ] すべての文に主語があるか
- [ ] 「これ」「上記」を具体的な言葉に書き換えたか
- [ ] 図の内容がすべてテキストで説明されているか
- [ ] 2,000文字を超えていないか（超えていれば分割）
- [ ] 臨時運用なら解除条件が書いてあるか

## Notion編集ルール（Notion使用時）

| ルール | 詳細 |
|--------|------|
| 見出し | H2=大見出し、H3=小見出し（H1は大きすぎるため使わない） |
| 色使い | 2色までに抑える |
| 赤字 | 絶対に忘れてはいけない操作のみ |
| トグル | 入れ子禁止。短い内容（2-3行）には使わない |
| フロー図 | Mermaid記法推奨 |
| ページ作成先 | プライベートではなくTeamspaceに公開（透明性重視） |

## ドキュメントフォーマット規則

### フォーマット優先順位

1. **テーブル** - 比較・一覧に最適
2. **箇条書き** - 列挙・手順に最適
3. **コードブロック** - 5行以内の例
4. **段落** - 最終手段（1行まで）

### 比較表の書き方

```markdown
| 避ける | 使う | 理由 |
|--------|------|------|
| 古い書き方 | 新しい書き方 | 1行の理由 |
```

### コード例のルール

| 項目 | 制限 |
|------|------|
| 行数 | 5行以内 |
| 説明 | 1行まで |
| コメント | 最小限 |

### セキュリティ

| 記載禁止 | 代替 |
|----------|------|
| APIキー | プレースホルダー `__API_KEY__` |
| パスワード | 例: `your-password` |
| 実URL | 例: `https://example.com` |
