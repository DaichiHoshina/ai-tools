# Notion 運用（AI・権限・外部連携）

## Notion AI活用

### AI自動入力プロパティ

- **AI Summary**: ページ内容から自動要約を生成（DBスキャン時に便利）
- **Custom AI Autofill**: プロンプト指定で自動分類・タグ付け・翻訳

### 効果的な使い方

| 用途 | AIプロパティ設定 | 例 |
|------|----------------|-----|
| 自動要約 | AI Summary | 議事録→3行要約 |
| 自動カテゴリ分類 | Custom Autofill「カテゴリをBackend/Frontend/Infraから選んで」 | ナレッジ自動分類 |
| キーワード抽出 | Custom Autofill「技術キーワードを3つ抽出」 | 検索性向上 |
| 翻訳 | Custom Autofill「タイトルを英語に翻訳」 | 多言語チーム |

### AIが苦手なこと

- 外部コンテキストが必要な判断（他DBの情報を参照できない）
- 複雑な多段推論
- 100%の精度が求められるプロパティ → 手動入力にする

## 権限・共有設計

### 基本原則

- **最小権限**: 必要最小限のアクセスのみ付与
- **親ページで制御**: 親ページの権限が子に継承される
- **ワークスペース権限 > チームスペース権限**: WSのフルアクセスはTSで制限不可

### チームスペース設計

| タイプ | 公開範囲 | 用途 |
|--------|---------|------|
| オープン | 全社員閲覧可 | 全社ナレッジ、社内Wiki |
| クローズド | メンバーのみ | 部署別ドキュメント |
| プライベート | 招待者のみ | 人事、経営、機密情報 |

### DB権限

- **can create pages**: 閲覧+行追加のみ（既存行の編集不可）
- 運用DBは「can create pages」、マスターDBは「can edit」を使い分ける

## 外部ツール連携

### リンクプレビュー（推奨）

| ツール | 表示内容 |
|--------|---------|
| GitHub PR/Issue | タイトル、ステータス、担当者 |
| Jira チケット | チケット名、ステータス、優先度 |
| Figma | デザインのライブプレビュー |
| Slack メッセージ | 投稿内容のスナップショット |
| Google Docs | ドキュメントプレビュー |

### 埋め込み vs リンクプレビュー

| 方式 | 用途 | 注意 |
|------|------|------|
| `/embed` | Figma、Miro等の操作可能な埋め込み | ページが重くなる |
| リンクプレビュー | 参照のみ（GitHub、Jira等） | 軽量、推奨 |
| メンションリンク | Notion内ページ参照 | ページタイトルで自動表示 |

- 1ページに埋め込みは**2つまで**（重くなるため）
- 参照目的ならリンクプレビューを優先

### Notion AI Connect

Slack、Google Drive、GitHub、Jira等の外部ソースをNotion AI検索に接続。ワークスペース横断で自然言語検索が可能。
