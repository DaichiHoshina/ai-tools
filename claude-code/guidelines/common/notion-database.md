# Notion データベース設計

## プロパティタイプの使い分け

| データの性質 | 使うプロパティ | 使わないもの | 理由 |
|-------------|--------------|-------------|------|
| カテゴリ・状態 | Select / Status | Text | 表記揺れ防止、フィルタ可能 |
| 複数タグ | Multi-select | Text（カンマ区切り） | 個別フィルタ・集計可能 |
| 日時 | Date | Text | カレンダービュー・ソート可能 |
| 金額・数量 | Number | Text | 合計・平均の自動計算可能 |
| 他テーブル参照 | Relation | Text（名前コピー） | Single Source of Truth |
| 集計値 | Rollup（Relation経由） | 手動入力 | 自動更新 |
| 計算・判定 | Formula | なし | 条件分岐・自動分類に使用 |
| 真偽値 | Checkbox | Select（Yes/No） | フィルタが直感的 |

## プロパティ命名規則

- **具体的な名前**にする（「Info」→「クライアント名」「公開日」）
- 英語: PascalCase（「PublishDate」「ClientBudget」）
- 日本語: 短く（「担当者」「期限」「優先度」）
- 同じ意味のプロパティ名をDB間で統一（「Status」に統一、「状態」「ステータス」混在禁止）

## Select / Multi-select 管理

- 選択肢は**事前定義**する（自由入力させない）
- 表記を統一（「進行中」と「In Progress」を混ぜない）
- 選択肢は10個以内。超えたらグルーピングか別プロパティに分割
- 色は意味を持たせる（赤=緊急、黄=注意、緑=完了 など）

## ビュー設計

| ビュータイプ | 用途 | 例 |
|-------------|------|-----|
| Table | 全件一覧・一括編集 | マスターデータ管理 |
| Board | ステータス別管理 | タスク管理（Todo/Doing/Done） |
| Calendar | 日付ベース管理 | リリーススケジュール |
| List | シンプルな一覧 | ナレッジ一覧 |
| Gallery | カード型表示 | ポートフォリオ・議事録 |
| Timeline | ガントチャート | プロジェクト進行 |

- 1つのDBに**目的別ビューを3〜5個**作る
- フィルタなしの「Master View」を1つ必ず残す
- ビューごとに表示プロパティを絞る（不要列は非表示に）

## ページ内テーブル表示

### シンプルテーブル vs データベース

| 用途 | 使うもの | 理由 |
|------|---------|------|
| 静的な比較表・仕様一覧 | シンプルテーブル | フィルタ不要、軽量 |
| ブレスト段階の整理 | シンプルテーブル | 構造が固まる前は柔軟な方がよい |
| フィルタ・ソートが必要 | データベース | 動的な絞り込みが可能 |
| 各行に詳細ページが必要 | データベース | 行をページとして展開可能 |
| 複数ビューで表示したい | データベース | Board/Calendar/Timeline切替 |

**判断基準**: 行数10行以下 + 更新頻度低い → シンプルテーブル。それ以外 → データベース。

### インラインデータベース

- **インライン**（ページ内埋め込み）をデフォルトにする
- 1ページにインラインDBは**3つまで**
- デフォルト表示行数は**10行**に制限

### 見やすいテーブルの作り方

- 列ヘッダー・行ヘッダーをONにする（背景色+太字で視認性向上）
- セル内テキスト折り返しを有効にする
- テーブル幅はページ幅に合わせる
- 表示プロパティは**5〜7列**に絞る

## パフォーマンス

- プロパティ数は**15個以内**（多すぎると表示が遅くなる）
- 完了済みレコードは定期的にArchiveビューに移動
- Formulaの入れ子は2階層まで

## テンプレート設計

### データベーステンプレート

- DBの「新規」ボタン横「▼」→ テンプレート追加で雛形を定義
- デフォルト値を設定（ステータス=Draft、日付=今日）
- 用途別に複数作成（「議事録」「ADR」「障害報告」等）

### ボタン活用

- **繰り返し作業はボタン化**（ページ作成+プロパティ設定を1クリック）
- アクション: ページ追加、プロパティ変更、ブロック挿入
- アイコン+短いラベルをつける（「＋ 議事録作成」「＋ 障害報告」）

### コンテンツ種別テンプレ

| 種別 | 必須プロパティ | ページ内構成 |
|------|--------------|------------|
| 議事録 | 日付, 参加者, プロジェクト | 目的 → アジェンダ → 討議内容 → 決定事項 → アクション |
| 1on1 | 日付, 相手, ステータス | 体調・近況 → 相談事項 → フィードバック → ネクストアクション |
| ADR | ADR番号, ステータス, カテゴリ | コンテキスト → 決定 → 代替案 → 結果 |
| 障害報告 | 日付, 重要度, ステータス | 概要 → タイムライン → 根本原因 → 影響範囲 → 再発防止 |
| 振り返り | 日付, スプリント | Keep → Problem → Try → アクション |

## アンチパターン

| やりがち | 正しいやり方 |
|---------|------------|
| Textに日付を書く | Date プロパティを使う |
| Textにカテゴリを書く | Select を使う |
| 同じ情報を複数DBに手動コピー | Relation + Rollup で参照 |
| 1つのDBに全種類のデータを詰め込む | 種類別にDBを分けてRelation |
| Selectに「その他」を入れる | 選択肢を見直すか、Text備考欄を別に用意 |
