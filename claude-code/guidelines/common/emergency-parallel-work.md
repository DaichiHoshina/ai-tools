# 緊急時対応・並行作業ガイド

## 緊急事態への対応

### デグレ発生時

| ステップ | 内容 |
|---------|------|
| 1. 動作確認 | 該当機能の動作確認 |
| 2. 影響範囲 | 影響範囲特定 |
| 3. 緊急性判定 | 本番影響有無を判断 |
| 4. 対応方針 | 修正 or ロールバック判断 |
| 5. リリース | テスト → レビュー → マージ |

### 「分からない」時

| ❌ 禁止 | ✅ 推奨 |
|---------|---------|
| 推測で実装 | 質問・相談する |
| 独断で進める | 類似コード参考 |
| 仕様無視 | ドキュメント再確認 |

## 並行作業・タスク分割

### グループ分け例

| グループ | 担当 | 依存関係 |
|---------|------|----------|
| **Group A（基盤）** | データ層・API、共通ライブラリ、認証基盤 | なし |
| **Group B（UI）** | フロントエンド、コンポーネント | A完了後 |
| **Group C（品質）** | リファクタ、テスト、ドキュメント | 並行可 |
| **Group D（機能）** | 新機能、複雑ロジック | A,B完了後 |

### 実行フェーズ

| Phase | 実行タスク | タイミング |
|-------|-----------|-----------|
| Phase 1 | A + C | 並行実行 |
| Phase 2 | B + D | 並行実行（Phase 1完了後） |
| Phase 3 | 統合テスト・デプロイ準備 | Phase 2完了後 |

---

## 注意点

| 項目 | 内容 |
|------|------|
| 依存関係 | 依存関係の明確化 |
| ファイル競合 | 同一ファイル同時編集回避 |
| 影響通知 | 他グループ影響タスクは完了通知必須 |
| 結合テスト | 各Phase完了時に結合テスト |
