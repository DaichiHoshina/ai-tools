# Notion デザインパターン

## カバー画像とアイコン

- **全ページにアイコンを設定**する
- アイコンはスタイルを統一（絵文字 or カスタムアイコン、混ぜない）
- カスタムアイコンは同一セットから選ぶ（推奨: [Phosphor Icons](https://phosphoricons.com/)、モノクロ統一）
- カバー画像はグラデーション or Unsplash写真。サイズは1500x600px推奨
- カバー画像の色味をページ内のアクセントカラーと合わせる

## 配色ルール

| 要素 | ルール |
|------|--------|
| アクセントカラー | 1色に絞る（青系 or 緑系等） |
| 背景色 | デフォルト白 + アクセント1色の淡い背景のみ |
| テキスト色 | 黒（デフォルト）+ グレー（補足）+ アクセント色（強調） |
| 赤 | 警告・最重要のみ。装飾には使わない |

- ページ内で使う色は**3色以内**（ベース + アクセント + 強調）
- DB内のSelect色もアクセントカラーに合わせて統一

## フォント

| フォント | 印象 | 用途 |
|---------|------|------|
| Default（サンセリフ） | モダン、クリーン | 一般ドキュメント |
| Serif | フォーマル、知的 | 長文、レポート |
| Mono | テクニカル、エンジニア向け | 技術ドキュメント |

- 1ページ内でフォントは**1種類**

## 余白とセクション区切り

- **H2見出しの前にディバイダー**（`/divider`）→ セクション間の視覚的分離
- 見出しとコンテンツの間に空行を入れない（Notionは自動で余白確保）
- 密集させすぎない。情報ブロック間に適度な空白を残す

## コールアウト

| パターン | アイコン | 背景色 | 用途 |
|---------|---------|--------|------|
| 概要 | 💡 | グレー背景 | ページ冒頭の要約 |
| 重要 | ⚠️ | 黄背景 | 注意事項 |
| 禁止 | 🚫 | 赤背景 | やってはいけないこと |
| ヒント | ✅ | 緑背景 | おすすめ、ベストプラクティス |

- 背景はデフォルト（グレー枠線のみ）が最も洗練される
- 1ページにコールアウトは**3つまで**

## トグル

- 補足情報・詳細手順をトグルに格納（本文をスッキリ保つ）
- タイトルは内容がわかる文にする（「詳細」→「認証フローの詳細手順」）
- **入れ子禁止**（トグル内トグルは迷子になる）
- 3行以下の内容はトグルにしない（そのまま表示）

## レイアウト

### マルチカラム

- `/2c` で2カラム、`/3c` で3カラム
- 左カラム=概要テーブル、右カラム=詳細説明が読みやすい
- カラム内のインラインDBは小さめビュー（List or Gallery）が合う
- 左右の**情報量を揃える**（片方だけ長いと見づらい）

### タブ付きレイアウト

- Relation経由で関連DBをタブ表示可能
- 用途: プロジェクト→タスク/議事録をタブ切替、クライアント→案件/連絡履歴をタブ切替
- タブ名は短く（「タスク」「メモ」「履歴」）
- タブ数は**4つまで**

### ダッシュボード/ホームページ

```
[カバー画像]
[アイコン + タイトル]

> 💡 コールアウト: このページの目的を1行で

--- (ディバイダー)

## セクション1        ## セクション2
[インラインDB]       [インラインDB]
（2カラムレイアウト）

--- (ディバイダー)

## リンク集
[よく使うページへのリンク一覧]
```

## よくあるデザインミス

| ミス | 改善 |
|------|------|
| 絵文字とカスタムアイコンの混在 | どちらかに統一 |
| 5色以上のカラフルなページ | 3色以内に抑える |
| コールアウトだらけ | 3つまで。重要度に差をつける |
| カバー画像なしのトップページ | 必ずカバーを設定 |
| 全文が同じサイズのテキスト | H2/H3で階層化、太字で強調 |
| ディバイダーなしの長いページ | H2の前にディバイダー |
