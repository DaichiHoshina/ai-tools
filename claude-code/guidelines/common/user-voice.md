# user voice — 対話型リライトガイド（NG辞書付き）

`/docs` で Notion 投稿前に適用。**「理解 > 表現」を確認する3層対話**で、ユーザー自身の言葉を draft に織り込む。

> `notion-writing.md` の文体ルール（主語必須等）は最終投稿時の必須ルール。本ガイドはそこに到達する前のリライト工程。両者で衝突する場合は `notion-writing.md` を優先。

## 質問方式一覧

| Layer | 質問方式 | ツール |
|-------|---------|-------|
| 1 Intent | 自由記述 | 通常メッセージで停止、応答待ち |
| 2 Understanding | 自由記述 | 通常メッセージで停止、応答待ち（説明不能時のみ AskUserQuestion） |
| 3 Expression | 選択式 | AskUserQuestion |

## 目的

1. **他者伝達性**: 意図が伝わる文章にする
2. **自己検閲**: 理解していない内容をそのまま提出しない

AI が書き換えるのではなく、**ユーザーの自由回答を draft に織り込む**。AI の生成物は素材、ユーザーの回答が一次ソース。

## 文体の基本

- 体言止め多用（「〜する」より「〜」で切る）
- 助詞最小（省略可なら省略）
- 数値で具体化（「3件」「128→96行」）
- 結論先出し
- 装飾語なし
- カタカナ技術用語はそのまま（「ワークフロー」「リポジトリ」等）

## 3層チェック

### Layer 1: Intent（意図確認）

**目的**: セクション主旨を言語化させ、主旨外の文を削除候補に。

- **発動条件**: セクション数 ≥ 3 **または** 本文 ≥ 300字
- **質問形式**: 自由記述（assistant が質問して会話停止、ユーザー応答待ち）
- **質問テンプレ**: 「このセクション「<見出し>」で読み手に何を伝えたい？一言で」
- **反映**: 主旨から外れる文を削除候補として提示、ユーザー確認後に削除

### Layer 2: Understanding（理解確認）

**目的**: 難解語・抽象表現をユーザーの言葉で再定義し、AI の表現を置換。

- **対象**: カタカナ6文字以上、3文字以上の英略語、業界固有用語
- **スキップ**: `user_vocabulary.md` の「用語定義」に登録済みの語
- **質問形式**: 自由記述（会話停止、ユーザー応答待ち）
- **質問テンプレ**: 「"<用語>" ここでどういう意味で使った？」
- **反映**:
  1. ユーザー回答文を draft 該当箇所に置換（AI が言い換えない、回答文を原文のまま使う）
  2. 用語と定義を `user_vocabulary.md` の「用語定義」に追記
- **説明できない場合**: 次のアクションを AskUserQuestion で確認 [調べる / 削除 / 曖昧なまま残す]

### Layer 3: Expression（表現確認）

**目的**: AI 定型・硬い文語を除去。

- **対象**: NG辞書ヒット（下記）
- **質問形式**: 選択式（`AskUserQuestion` 使用）
- **質問テンプレ**: 「"<語>" AI っぽい？」[そのまま / 削除 / 書き換え]
- **反映**: 選択に応じて draft を更新、回答を `user_vocabulary.md` に蓄積

#### AI定型 NG辞書

```
効果的に / 効率的に / シームレスに / 直感的に / 革新的な / 素晴らしい / 強力な
より良い / 〜を実現します / 〜を提供します / 〜を可能にします
〜することができます / ご紹介します / ご覧ください / 〜いただけます
ユーザーエクスペリエンス / ユーザビリティ向上
まず〜しましょう / 重要なポイント / 注目すべき点
本機能は / 本ドキュメントは / 本記事では
```

#### 硬い文語 NG辞書

```
〜である / 〜であった / 〜されている / 〜となる
〜に基づき / 〜に関して / 〜について述べる
当該 / 当方 / 貴殿 / 以下に示す
```

## トリアージルール（質問過多回避）

**合計上限 9件**（3+3+3）。

| Layer | 上限 | 省略条件 |
|-------|------|---------|
| 1 Intent | 3件（セクション） | 本文<300字 または セクション<3 |
| 2 Understanding | 3件（難解語） | 既知語 ≥ 3件、または難解語 < 3 |
| 3 Expression | 3件（NG辞書ヒット） | ヒット 0件 |

上位絞り込み: 出現頻度順。既知語は常にスキップ。

## user_vocabulary.md の形式

auto-memory `~/.claude/projects/{project}/memory/user_vocabulary.md` に蓄積。

```markdown
## 用語定義（ユーザーの言葉）
# 形式: 用語 — 定義（YYYY-MM-DD）
- オーケストレーション — 複数エージェントを順番に動かす仕組み（2026-04-21）

## セクション主旨（意図の型）
# 形式: セクション種別 — よく使う主旨表現
- アーキテクチャ — 「何と何がどうつながるか」

## 嫌う表現
# 形式: 元表現 → 処理
- 効果的に → 削除
- ご紹介します → 削除

## 好む言い換え
# 形式: 元表現 → 置換表現
- 実装する → 書く
- 確認する → 見る
```

## Before/After 変換例

| Before（AI調） | After（user調） |
|--------------|------------------|
| 本機能は、効率的にドキュメントを作成することを可能にします | ドキュメント作成支援 |
| まずは、以下の手順を確認しましょう | 手順: |
| ユーザーエクスペリエンスの向上を実現します | UX改善 |
| 本記事では、〜について述べる | 〜について |
| 該当する3つのポイントがあります | 3点 |
| 効果的に活用いただけます | 使える |

## /docs との接続

```
Step 5 draft 完成
  ↓
Step 5.5 対話型リライト（本ガイド）
  - Layer 1 → Layer 2 → Layer 3 の順
  - Layer 2 のユーザー回答文は draft にそのまま織り込む（AIで言い換えない）
  - 回答は user_vocabulary.md に追記
  ↓
Step 6 投稿
```
