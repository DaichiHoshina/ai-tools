# /memory-save - Quick Serena memory save

Save work context to Serena memory in 1 command.

## Flow

1. **Auto-generate save content**
   - current task/work
   - progress status
   - critical context (file paths, design decisions)
   - next action

2. **Auto-determine memory name**
   - w/ arg → use specified name
   - no arg → auto-generate `work-context-YYYYMMDD-HHMMSS`

3. **Save**
   ```
   mcp__serena__write_memory(
     memory_name: "<name>",
     content: "<auto-generated content>"
   )
   ```

4. **Confirm display**
   - show saved memory name & summary

## Options

| Arg | Description | Example |
|-----|-------------|---------|
| (none) | auto-name save | `/memory-save` |
| `<name>` | specify memory name | `/memory-save auth-refactor-progress` |

## When to use

- mid-point save for long work
- manual backup before compact
- hand-off to next session

## Fallback behavior

| Scenario | Action |
|----------|--------|
| serena connection fails | save to local `~/.claude/memory-fallback/{name}.md`, warn |
| write fails (permission etc) | output content to chat, guide user on manual save |
| memory name collision | auto-suffix `-2`, `-3` |
