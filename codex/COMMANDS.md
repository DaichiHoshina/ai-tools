# Codex コマンド一覧

> ai-toolsリポジトリの`claude-code/commands/`からコマンドが利用可能です。

## 主要コマンド

### ワークフロー系

| コマンド | 説明 | 使用例 | 頻度 |
|---------|------|--------|------|
| `/flow` | タスク自動判定・実行（万能コマンド） | 迷ったらこれを使用 | ⭐⭐⭐⭐⭐ |
| `/dev` | 実装用コマンド（直接実装） | 実装が明確な場合 | ⭐⭐⭐⭐⭐ |
| `/plan` | 設計・計画用（PO Agent使用） | 複雑なタスクの設計 | ⭐⭐⭐ |

### レビュー系

| コマンド | 説明 | 使用例 | 頻度 |
|---------|------|--------|------|
| `/review` | 状況に応じた適切なレビュー | 実装完了後の確認 | ⭐⭐⭐⭐ |
| `/code-quality-review` | アーキテクチャ・コード品質 | 設計レビュー | ⭐⭐⭐ |
| `/security-error-review` | セキュリティ・エラーハンドリング | セキュリティチェック | ⭐⭐⭐ |

### Git統合

| コマンド | 説明 | 使用例 | 頻度 |
|---------|------|--------|------|
| `/commit-push-pr` | コミット→プッシュ→PR作成 | 一括Git処理 | ⭐⭐⭐⭐⭐ |
| `/commit` | コミットヘルパー | 差分から適切なメッセージ生成 | ⭐⭐⭐⭐ |

### 開発支援

| コマンド | 説明 | 使用例 | 頻度 |
|---------|------|--------|------|
| `/brainstorm` | 対話的設計精緻化 | 要件整理・設計相談 | ⭐⭐⭐⭐ |
| `/tdd` | TDD開発モード | RED-GREEN-REFACTOR強制 | ⭐⭐⭐ |
| `/test` | テスト作成専用 | 既存コードのテスト作成 | ⭐⭐⭐ |
| `/debug` | デバッグ支援 | エラー原因特定 | ⭐⭐⭐ |
| `/refactor` | リファクタリング | コード改善（自動ガイドライン読込） | ⭐⭐⭐ |

## エイリアス（短縮コマンド）

| エイリアス | フルコマンド |
|-----------|-------------|
| `/cpr` | `/commit-push-pr` |
| `/rv` | `/review` |
| `/tdd` | `/superpowers:test-driven-development` |
| `/dbg` | `/superpowers:systematic-debugging` |

## コマンド選択ガイド

```
迷ったら         → /flow（タスク自動判定）
実装明確         → /dev
設計から         → /plan
レビュー         → /review
一括Git処理      → /commit-push-pr（エイリアス: /cpr）
ブレスト・相談   → /brainstorm（エイリアス: /br）
TDD開発          → /tdd
デバッグ         → /debug
```

## 使用例

### 新機能実装
```
1. /brainstorm で要件整理
2. /plan で設計（複雑な場合）
3. /dev で実装（または /tdd でTDD開発）
4. /review でレビュー
5. /cpr でコミット→PR作成
```

### バグ修正
```
1. /debug でエラー解析
2. /dev で修正実装
3. /test でテスト追加
4. /cpr でコミット→PR作成
```

### リファクタリング
```
1. /refactor でリファクタリング（ガイドライン自動適用）
2. /review で品質確認
3. /cpr でコミット→PR作成
```

## 詳細

完全なコマンドリストは`~/ai-tools/claude-code/commands/`を参照してください。

**Note**: コマンドはシンボリックリンクを通じて利用可能になります（install.sh実行後）。
